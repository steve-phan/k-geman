# K-Geman Android Project - Build Instructions

## Important Note
This project requires Android development environment with internet access to download dependencies.

## Build Requirements
- **Android Studio**: Hedgehog (2023.1.1) or later
- **JDK**: 17
- **Gradle**: 8.2 (wrapper included)
- **Android SDK**: API 24-34
- **Internet connection**: Required for first build to download dependencies

## First-Time Setup

### 1. Install Android Studio
Download from: https://developer.android.com/studio

### 2. Configure Android SDK
- Open Android Studio > Settings > Android SDK
- Install SDK Platform 34 (Android 14)
- Install SDK Platform 24 (Android 7.0) for minimum support
- Install Android SDK Build-Tools 34.0.0

### 3. Firebase Configuration
Replace `app/google-services.json` with your actual Firebase configuration file:
1. Go to [Firebase Console](https://console.firebase.google.com)
2. Create a new project or use existing
3. Add Android app with package name: `com.kgeman`
4. Download `google-services.json`
5. Replace the placeholder file in `app/` directory

### 4. Enable Firestore
1. In Firebase Console, go to Firestore Database
2. Create database in test mode
3. Deploy security rules: `firebase deploy --only firestore:rules`

### 5. Build the Project
```bash
# From project root
./gradlew assembleDebug

# Or use Android Studio
# File > Open > Select k-geman directory
# Build > Make Project
```

## Dependency Download
On first build, Gradle will download:
- Android Gradle Plugin 8.2.0
- Kotlin 1.9.20
- Firebase BOM 32.7.0
- Room 2.6.1
- AndroidX libraries
- Material Components
- WorkManager
- Coroutines

Total download size: ~500-700 MB

## Build Time
- First build: 5-10 minutes (downloading dependencies)
- Subsequent builds: 30-60 seconds

## Common Issues

### Issue: "No address associated with hostname"
**Solution**: Check internet connection. Gradle needs to download dependencies from:
- https://dl.google.com
- https://repo1.maven.org
- https://plugins.gradle.org

### Issue: "SDK location not found"
**Solution**: Create `local.properties` in project root:
```properties
sdk.dir=/path/to/Android/Sdk
```

### Issue: "KSP plugin not found"
**Solution**: Ensure you're using Gradle 8.2 or later. Run:
```bash
./gradlew wrapper --gradle-version=8.2
```

## Verify Build Success
```bash
./gradlew assembleDebug
# Look for: BUILD SUCCESSFUL in XXs

# Check generated APK
ls -lh app/build/outputs/apk/debug/app-debug.apk
```

## Running the App

### On Emulator
```bash
./gradlew installDebug
```

### On Physical Device
1. Enable Developer Options on device
2. Enable USB Debugging
3. Connect device via USB
4. Run: `./gradlew installDebug`

## Gradle Wrapper Files
The following files will be generated by Gradle wrapper command:
- `gradlew` (Unix/Mac)
- `gradlew.bat` (Windows)
- `gradle/wrapper/gradle-wrapper.jar`
- `gradle/wrapper/gradle-wrapper.properties`

These files are necessary for building the project but are not included in this submission due to environment limitations.

To generate them in a proper environment:
```bash
gradle wrapper --gradle-version=8.2
```
